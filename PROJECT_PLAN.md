# Project Plan

## 목표

문제 생성 기능과 교육적 설명을 연결해 학생, 보호자, 교사가 개념 이해부터 반복 연습과 출력까지 한 흐름에서 수행하도록 합니다.

## 설계 원칙

1. 실제 문제를 저장하거나 기존 교재에서 복사하지 않고 생성 규칙을 저장합니다.
2. UI는 Problem Engine을 경유하며 Generator를 직접 호출하지 않습니다.
3. 학습 콘텐츠는 `src/data/learningContent`에 두어 문제 생성 데이터와 분리합니다.
4. 구현되고 검토된 범위만 목록, 내부 링크와 sitemap에 노출합니다.
5. 광고 승인을 위한 반복 문장이나 빈 페이지를 만들지 않습니다.
6. 정적 내보내기와 모바일·키보드 탐색을 유지합니다.

## 현재 완료 범위

- 초등 5학년 분수 덧셈 유형 3개 생성
- 세 유형의 고유 학습 콘텐츠와 단계별 예제
- 학습 전체 목록 및 분수 단원 목록
- 콘텐츠→생성기 query parameter 연결
- 소개, 개인정보처리방침, 이용약관, 문의 페이지
- 페이지별 SEO metadata, canonical, Open Graph
- robots 및 실제 경로만 포함하는 sitemap
- 학습 상세 BreadcrumbList 구조화 데이터

## 다음 단계

- 기존 세 콘텐츠의 교육 현장 피드백 반영
- 접근성 및 실제 기기 회귀 테스트 자동화
- Problem Engine의 검증 범위를 유지하며 다음 한 개 단원을 작은 범위로 추가
- 배포 환경별 canonical 도메인을 설정으로 관리하는 절차 확립
